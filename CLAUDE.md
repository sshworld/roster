# CLAUDE.md

Guidance for Claude Code (and other AI coding agents) working in this repository.
Human contributor workflow lives in [CONTRIBUTING.md](CONTRIBUTING.md) — both apply.

## Project Overview

roster is a **zero-runtime-dependency** TypeScript CLI (npm `roster-cli`) plus a
Claude Code plugin wrapper. It statically audits AI agent rosters: description
overlap (TF-IDF), missing tool declarations, routing ambiguity, context cost,
and joins real invocation history from Claude Code transcripts.

- `src/` — CLI source. ESM, Node >= 20, built with plain `tsc` (no bundler).
- `src/sources/` — roster loaders (dir, user, plugin-cache, github). Sweep
  sources (dir/github) ingest only agent-shaped markdown: explicit frontmatter
  `name`, basename not `SKILL.md`/`CLAUDE.md`/`AGENTS.md`.
- `src/rules/` — audit rules (overlap, harness, routing, cost, fluff).
- `skills/`, `hooks/`, `.claude-plugin/` — the Claude Code plugin layer.
- `tests/` — vitest. `scripts/bench.sh` — SHA-pinned benchmark suite.

## Commands

```bash
npm run build        # tsc → dist/
npm test             # vitest (offline — must pass with no network)
npm run demo:report  # regenerate docs/demo/report.html from fixtures
scripts/bench.sh     # regenerate docs/benchmarks/ (network, SHA-pinned)
```

## Hard Rules

- **Zero runtime dependencies.** Never add a `dependencies` entry. Node
  builtins only (`node:fs`, `node:readline`, ...). PRs adding a runtime dep
  are rejected outright; dev-dependencies need a strong reason.
- **Tests are offline.** Anything touching the network (GitHub API, raw
  fetches) is mocked (`globalThis.fetch` stubs — see
  `tests/unit/sources-github.test.ts`). Never call live endpoints in tests.
- **Behavior changes require tests** (Red → Green preferred). CI gates on
  build + tests (Node 20/22) + the bash hook tests.
- **stdout is data, stderr is diagnostics.** `--json` output must stay pure
  parseable JSON; the MCP server (`src/mcp.ts`) additionally requires stdout
  to carry only JSON-RPC lines.
- Commit messages: Conventional Commits, no scope (`fix: ...`, `feat: ...`).
- Benchmarks are SHA-pinned in `scripts/bench.sh` — never "update the numbers"
  by hand; re-run the script. README leaderboard table is generated by
  `scripts/update-bench-table.mjs` between the `bench:start`/`bench:end`
  markers — do not edit it manually.

## Gotchas

- Running `npx roster-cli` from inside this repo resolves to the local project
  (package-name match) and fails with `roster: command not found` — test npx
  behavior from any other directory.
- The plugin installer runs `npm install` but ignores lifecycle scripts, so
  `dist/` cannot be built via `prepare`; the roster-audit skill self-heals by
  building on first run.
- Releases (version bump in 3 files, `v*` tag → CI npm publish, manual GitHub
  release) are maintainer-only — do not tag or bump in PRs.
